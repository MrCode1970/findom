# Movements Table Schema

One row equals one normalized movement (never a two-sided transaction row).

| Column | Type | Required | Description |
| --- | --- | --- | --- |
| provider | text | yes | Connector name. |
| external_id | text | yes | Stable idempotency key generated by connector. |
| account_id | text | yes | Account owner of this movement side. |
| occurred_at | timestamp with time zone | yes | Movement timestamp in ISO-8601 format. |
| amount | decimal(18,2) | yes | Signed amount: debit is negative, credit is positive. |
| currency | text | yes | ISO 4217 code. |
| merchant | text | no | Clean merchant name. |
| comment | text | no | Clean user/provider comment. |
| transfer_id | text | no | Shared id for both sides of transfer (if known). |
| meta | json | no | Additional raw details needed for troubleshooting. |

Primary key recommendation: `(provider, external_id)`.
Foreign key recommendation: `(provider, account_id)` -> accounts table.
